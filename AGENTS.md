# AGENTS.md — правила для агента, работающего над LifeOss

Этот файл читается ПЕРВЫМ любым агентом (Claude Code, Codex, OpenCode, PromptQL
и др.), который берётся за проект. Он задаёт стиль общения и даёт краткую
сводку. Подробности — в `docs/agent-handoff.md`.

## Стиль общения (обязательно)

1. **Только русский язык.** Владелец общается по-русски — отвечай по-русски.
2. **Живо и дружелюбно, но по делу.** Никакого канцелярита и воды.
3. **Эмодзи — да.** Умеренно, для акцентов и настроения: ✅ 🚀 🎨 ⏳ 💪 и т.п.
   В заголовках списков и статусах — приветствуется.
4. **Структура.** Ответы форматируй: заголовки, короткие списки, `код` в
   бэктиках. Большие ответы дели на блоки. Команды для владельца — отдельным
   блоком кода (например `git pull` → `npm run tauri dev`).
5. **Итог в конце.** После работы: что сделано (кратко, списком), ссылка на
   коммит, что владельцу сделать у себя, и вопрос/предложение следующего шага.
6. **Не переспрашивай лишнего.** Владелец пишет пачку правок — делай всё
   пачкой, целыми модулями. Спрашивай только если реально неоднозначно.
7. **Всё, что владелец написал в чате — истина.** Каждое его решение фиксируй
   в `docs/decisions-log.md` в той же сессии.

## Короткая сводка проекта

- **LifeOss** — персональная local-first Windows-десктоп «ОС жизни» для одного
  владельца (werty4870 / GitHub sigmuchiii). Не публичный продукт. UI — русский.
- **Стек:** Tauri 2 + React 19 + TypeScript + Vite; Rust workspace
  (`lifeos-core`, `lifeos-storage`); SQLite (WAL) + миграции.
- **Дизайн:** Bento v3 (`docs/design/bento-design.html`) — светлая по умолчанию,
  5 тем в Настройках (`html[data-theme]`), акцент #6c5ce7, Inter + JetBrains
  Mono, Ctrl+K палитра. Сайдбар — иконочный рейл 64px с тултипами `data-tip`.
  Поля ввода — класс `.field`; выбор даты — общий `DatePicker`.
- **Готово (2026-07-04):** Задачи (TickTick-стиль), Сегодня, Привычки,
  Календарь, Фокус, Быстрые заметки, Настройки. Заглушки: Дневник, Здоровье,
  Проекты, Заметки, Контакты, Финансы, Трейдинг.
- **Workflow:** агент коммитит в `main` через GitHub API; владелец собирает
  локально (`git pull` → `npm run tauri dev`) и присылает правки/скриншоты.
  Агент сам собрать приложение не может — проверяй код до коммита.
- **Порядок чтения доков:** этот файл → `docs/agent-handoff.md` →
  `docs/decisions-log.md` (свежие записи!) → спека модуля в `docs/modules/` →
  `docs/open-questions.md` (отложено, не решать самовольно).

## HTML-превью приложения (обязательно)

- В `docs/design/app-preview.html` лежит статичный HTML-вариант приложения:
  все экраны, темы, актуальные стили (иконки — эмодзи, данные — примерные).
- **При каждом изменении UI обновляй этот файл в том же коммите.** Он всегда
  должен отражать текущее состояние приложения.
